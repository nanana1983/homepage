# Web Project – Timeline & Portfolio Dashboard

## 프로젝트 개요
본 프로젝트는 웹 응용에서 데이터베이스의 역할(저장·조회·관리)을 이해하기 위해 제작한 개인 웹 프로젝트이다.  
웹 페이지의 시각적 요소보다는 **사용자 입력 데이터의 구조적 저장과 재사용**에 중점을 두었으며,  
브라우저 환경에서 동작하는 **LocalStorage 기반 데이터 관리 방식**을 활용하였다.

---

## 프로젝트 목적
- 웹 응용에서 데이터가 어떻게 저장되고 유지되는지 이해
- 페이지 간 데이터 공유 및 재사용 구조 학습
- 데이터베이스 개념(영속성, 분리 저장, 조회)을 실제 코드로 구현

---

## 사용 기술
- **HTML / CSS**
- **Vanilla JavaScript**
- **Web Storage API (LocalStorage)**

---

## 데이터베이스 설계 개요
본 프로젝트는 별도의 DBMS를 사용하지 않고, 브라우저의 LocalStorage를 데이터 저장소로 사용한다.  
데이터의 성격에 따라 서로 다른 key를 사용하여 저장 구조를 분리하였다.

| 데이터 종류 | LocalStorage Key | 설명 |
|------------|------------------|------|
| Timeline   | `timeline`       | 방명록 형식의 사용자 글 저장 |
| Portfolio | `portfolio`      | 포트폴리오 게시글 저장 |

- 페이지 새로고침 및 브라우저 종료 후에도 데이터 유지
- 각 데이터는 독립적으로 관리
- 홈 화면에서는 최근 데이터 일부만 조회

---

## 페이지 구성 및 기능

###  Home (index.html)
- Timeline / Portfolio의 최근 게시글 요약 표시
- LocalStorage에 저장된 데이터를 읽어와 렌더링
- 메인 대시보드 역할

###  Timeline (timeline.html)
- 사용자 방명록 작성 및 목록 표시
- 글 작성 시 LocalStorage에 즉시 저장
- 최신 글이 상단에 표시되도록 구현

###  Portfolio (portfolio.html)
- 포트폴리오 게시글 작성
- 텍스트 및 파일(이미지) 저장 가능
- LocalStorage를 이용한 게시글 관리

---

##  프로젝트 특징
- 데이터의 **영속성(Persistence)** 확보
- 페이지 간 **데이터 공유 구조** 구현
- 데이터 저장 → 조회 → 화면 출력의 전체 흐름을 순수 JavaScript로 구현
- 별도 서버 없이 클라이언트 환경에서 데이터베이스 개념 학습 가능
